git常用命令

Yang Jian
2024-07-20
card
git
  1. git init 初始化新仓库
  2. git add [file/.] 添加新文件/所有文件到暂存
  3. git commit -m "xxx" 提交修改到本地版本库并附加提交信息
    1. git commit -F file 使用文件内容作为提交信息
    2. git commit --reuse-message 使用之前某次（由 commit-id 确定）提交的 message 作为本次提交的 message
      • -C <commit> 作用同上
  4. git commit --reedit-message= 作用同上，但是调用编辑器允许修改提交信息
    • -c <commit> 作用同上
  5. git commit --amend 修改上次的提交信息
  6. git push 同步到远程仓库，默认推送本地分支到有跟踪关系（通常是同名）的远程分支
    • git push [-f] origin a:b 将本地 a 分支的内容推送到远程 b 分支，如果 b 分支有不同的提交历史，需要使用 -f 选项强制覆盖
  7. git pull 同步远程仓库到本地
  8. git clone https://git.xxxx 复制远程仓库到本地
    1. 如果远程仓库中还包含子仓库，则需要使用 git submodule update --init --recursive 下载子模块
    2. 或者 git clone --recursive https://xxxx.git 在克隆远程仓库时将包含的子仓库也克隆下来
    3. git clone xxxx --depth=1 只拷贝最新的提交，不下载历史记录
  9. git status 查看本地仓库状态
  10. git log [-N] 查看仓库提交日志
    • git log [-p] filename 查看单个文件的修改记录，-p 选项查看具体的修改内容
  11. git diff 查看所有改动文件的与上次提交时的差异
  12. git restore [file] 放弃文件修改
